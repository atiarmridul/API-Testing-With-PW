# Contributing Guide

This project is a Playwright test suite with markdown docs automation.

## Scope
- Test files live in `tests/` and should use the `.spec.ts` suffix.
- Keep non-root markdown files in `docs/`.
- Keep only `readme.md` and `agent.md` at repository root.

## Local Setup
1. Install dependencies:
   - `npm install`
2. Install git hooks (once per local clone):
   - `bash scripts/install-hooks.sh`
3. Optional live docs sync while editing markdown:
   - `bash scripts/watch-markdown.sh`

## Development Workflow
1. Create/update tests in `tests/`.
2. Run tests locally:
   - `npx playwright test`
3. Update docs when behavior or workflow changes:
   - `readme.md`
   - `agent.md`
   - `docs/changelog.md`
   - `docs/agent-progress.md`
4. Sync markdown index if needed:
   - `bash scripts/sync-markdown.sh`
5. Commit and push.

## Commit Checklist
- Tests pass locally (or failures are clearly understood).
- `docs/index.md` is current (auto-generated by hook/script).
- `docs/changelog.md` includes notable updates.
- `docs/agent-progress.md` has a new session entry for meaningful changes.
